# Git — Полное руководство

## Что такое Git?

**Git** — это **распределённая система контроля версий** (Distributed Version Control System), созданная **Линусом Торвальдсом** в 2005 году для разработки ядра Linux.

Git позволяет:
- Отслеживать изменения в коде
- Возвращаться к предыдущим версиям
- Работать в команде без конфликтов
- Создавать ветки для экспериментов

**Отличительная особенность**: каждый разработчик имеет **полную копию** репозитория на своём компьютере.

---

## Основные понятия Git

- **Repository (Репозиторий)** — хранилище проекта
- **Commit** — снимок состояния проекта в определённый момент
- **Branch (Ветка)** — независимая линия разработки
- **Remote** — удалённый репозиторий (GitHub, GitLab и т.д.)
- **Merge** — объединение изменений из разных веток
- **Pull Request (Merge Request)** — предложение изменений

---

## Виды Git-репозиториев

| Вид                    | Описание                                      | Когда использовать                     |
|------------------------|-----------------------------------------------|----------------------------------------|
| **Bare Repository**    | Только служебные файлы (без рабочей копии)    | Центральный сервер                     |
| **Non-bare**           | Обычный репозиторий с рабочими файлами       | Локальная разработка                   |
| **Monorepo**           | Один большой репозиторий                     | Крупные проекты с множеством модулей  |
| **Multi-repo**         | Много маленьких репозиториев                 | Микросервисы                           |
| **Fork**               | Копия чужого репозитория                     | Внесение изменений в open-source       |

## Сравнение Git с другими системами

| Критерий               | Git                     | SVN                     | Mercurial             |
|------------------------|-------------------------|-------------------------|-----------------------|
| Тип                    | Распределённый          | Централизованный        | Распределённый        |
| Скорость               | Очень высокая           | Средняя                 | Высокая               |
| Работа без интернета   | Полная                  | Ограниченная            | Полная                |
| Ветвление              | Лёгкое и быстрое        | Медленное               | Хорошее               |
| Популярность           | №1 в мире               | Устаревает              | Низкая                |
| GitHub/GitLab          | Отличная интеграция     | Слабая                  | Средняя               |

## Плюсы Git

1. **Полная история** проекта у каждого разработчика
2. **Быстрое ветвление** и слияние
3. **Отличная работа в оффлайне**
4. **Мощные инструменты** (rebase, cherry-pick, bisect)
5. **Огромная экосистема** (GitHub, GitLab, VS Code)
6. **Безопасность** — сложно потерять данные
7. **Бесплатный** и open-source

---

## Минусы Git

1. **Крутая кривая обучения** для новичков
2. **Сложные ситуации** (merge conflicts, rebase hell)
3. **Размер репозитория** растёт со временем
4. **Проблемы с большими бинарными файлами** (решается Git LFS)
5. **Можно случайно потерять изменения** при неправильных командах

**`.vscode/settings.json`**:
```json
{
    "git.enableSmartCommit": true,
    "git.autofetch": true,
    "git.confirmSync": false,
    "git.ignoreSubmodules": true,

```
## Примеры команд Git

**1. Начало работы**
```Bash

# Инициализация репозитория
git init

# Клонирование проекта
git clone 

# Добавление удалённого репозитория
git remote add origin 
```
**2. Ежедневный workflow**
```Bash

# Просмотр статуса
git status

# Добавление файлов
git add .
git add boiler_model.py

# Коммит
git commit -m "feat: улучшена модель бойлера с учётом тепловых потерь"

# Отправка изменений
git push origin main

# Получение изменений
git pull origin main
```
**3. Работа с ветками**
```Bash

# Создание новой ветки
git checkout -b 

# Просмотр всех веток
git branch -a

# Переключение на ветку
git checkout main

# Слияние ветки
git merge 
```
**4. Полезные команды**
```Bash

# Отмена последнего коммита
git reset --soft HEAD~1

# Просмотр истории
git log --oneline --graph --all

# Сравнение изменений
git diff

# Возврат файла к предыдущему состоянию
git checkout -- boiler_model.py

# Удаление файла из индекса
git rm --cached large_file.bin

```

## Полезные расширения VS Code для Git

1. **GitLens** — лучший инструмент (история строк, blame, сравнение)
2. **Git Graph** — визуализация веток
3. **GitHub Pull Requests** — работа с PR прямо в VS Code
4. **Conventional Commits** — подсказки сообщений коммитов
