# GitHub Flow — Полное руководство

## Что такое GitHub Flow?

**GitHub Flow** — это **лёгкий и эффективный** рабочий процесс (workflow) с использованием Git, специально разработанный компанией GitHub для быстрой и безопасной разработки.

Это **упрощённая версия Git Flow**, которая идеально подходит для современных команд, работающих с непрерывной интеграцией (CI/CD).

**Основная идея**: 
- `main` (или `master`) — всегда рабочая и готов к продакшену ветка.
- Все изменения делаются через **короткоживущие ветки** и **Pull Requests**.

---

## Основные принципы GitHub Flow

1. **main** — всегда стабильная и деплоится
2. Всё, что попадает в `main`, должно быть **готово к релизу**
3. Новые функции/исправления создаются в отдельных ветках
4. Используются **Pull Request** для ревью кода
5. После одобрения — merge в `main` + удаление ветки

---

## Сравнение GitHub Flow и Git Flow

| Критерий                    | GitHub Flow                        | Git Flow                              |
|----------------------------|------------------------------------|---------------------------------------|
| Количество веток           | Минимальное                        | Много (develop, release, hotfix)      |
| Ветка разработки           | `main`                             | `develop`                             |
| Подходит для               | Web, DevOps, небольшие команды     | Крупные enterprise-проекты            |
| Частота релизов            | Очень частая (Continuous Delivery) | Реже                                  |
| Сложность                  | Низкая                             | Высокая                               |
| Pull Requests              | Обязательны                        | Желательны                            |
| Hotfix                     | Прямо из feature-ветки             | Отдельная ветка `hotfix`              |

**Вывод**: GitHub Flow лучше подходит для вашего учебного/практического проекта.

---

## Плюсы GitHub Flow

1. **Простота** и лёгкость освоения
2. **Быстрый цикл** разработки
3. **Непрерывная интеграция** (CI/CD)
4. **Чистая история** в `main`
5. **Отличная поддержка** в GitHub (Pull Requests, Reviews, Checks)
6. **Меньше конфликтов** слияния
7. **Идеально** работает с VS Code + GitHub

---

## Минусы GitHub Flow

1. **Не очень подходит** для крупных проектов с множеством версий
2. **Меньше контроля** над этапами релиза
3. **main** может быть нестабильным при частых изменениях
4. Требует дисциплины от команды

---

## Полный Workflow GitHub Flow

### 1. Клонирование репозитория

```bash
git clone 
cd practice_project
git checkout main
git pull origin main
```
### 2. Создание новой ветки
```bash

# Для новой функциональности
git checkout -b feature/opc-ua-client-improvements

# Для исправления ошибки
git checkout -b bugfix/fix-boiler-temperature-calculation

# Для документации
git checkout -b docs/update-models-guide
```
### 3. Работа над задачей
```bash

# Редактируете файлы...
git add .
git commit -m "feat(opcua): улучшена обработка подключения клиента"
git commit -m "refactor(models): вынесены константы в класс BoilerModel"

# Отправка изменений
git push origin
```
