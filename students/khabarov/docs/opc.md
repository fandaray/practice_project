# OPC (OLE for Process Control) — Полное руководство

## Что такое OPC?

**OPC** (Open Platform Communications, ранее — OLE for Process Control) — это набор стандартов и технологий для **единого интерфейса обмена данными** между промышленными устройствами, контроллерами и программными системами (SCADA, MES, HMI и др.).

OPC состоит из **OPC-сервера** (предоставляет данные) и **OPC-клиента** (получает данные).

---

## OPC DA (Data Access) — Классический стандарт

### Общая информация
OPC DA — наиболее распространённый стандарт обмена данными **в реальном времени**.  
Данные в OPC представляются в виде **тегов** (переменных) с определёнными свойствами: значение, качество, временная метка.

### Режимы чтения данных:
1. **Синхронный** — клиент ждёт ответа
2. **Асинхронный** — клиент отправляет запрос и продолжает работу
3. **Подписка (Data Change)** — сервер сам уведомляет клиента об изменениях
4. **Режим обновления** — клиент запрашивает только "активные" теги

### Недостатки OPC DA
- Полная зависимость от **Windows + DCOM**
- Сложная настройка DCOM-безопасности
- Проблемы с межсетевым взаимодействием (брандмауэры)
- **Слабая безопасность** (нет шифрования)
- Нет поддержки истории данных (только текущие значения)
- Плохая масштабируемость (> 10 000 тегов)

## Стандарт OPC UA (Unified Architecture)

### Что такое OPC UA?

**OPC UA** — современный, кросс-платформенный стандарт, разработанный для устранения недостатков OPC DA.  
Это не просто протокол, а **полноценная архитектура**, включающая:
- Объектно-ориентированную информационную модель
- Встроенную безопасность
- Несколько транспортных протоколов
- Поддержку PubSub

### Преимущества OPC UA

1. **Кросс-платформенность** (Windows, Linux, macOS, embedded)
2. **Встроенная безопасность** (TLS, сертификаты X.509, OAuth, шифрование)
3. **Единая модель данных** (объекты, переменные, методы, ссылки)
4. **Разнообразие транспорта**: OPC UA TCP, HTTPS, MQTT, PubSub, TSN
5. **Надёжность и отказоустойчивость**
6. **Масштабируемость** (от датчиков до облака)
7. **Открытость** (стандарт IEC 62541)

### Сравнение OPC UA и OPC DA

| Характеристика          | OPC UA                          | OPC DA                     |
|-------------------------|---------------------------------|----------------------------|
| Платформа               | Кросс-платформенный             | Только Windows             |
| Безопасность            | TLS, сертификаты, шифрование    | Нет (DCOM уязвим)          |
| Модель данных           | Объектно-ориентированная        | Плоская (теги)             |
| Сеть                    | NAT/брандмауэры — нормально     | Проблемы с DCOM            |
| Реальное время          | Оптимизированные подписки       | Быстрее, но небезопасно    |
| История данных          | Есть (HDA)                      | Нет                        |
| IoT / Облако            | Отличная поддержка              | Практически отсутствует    |

## Теги в OPC

### В OPC DA:
Тег — это **предопределённая переменная** с именем, типом, доступом.

### В OPC UA:
Тег = **Node** (узел) в Address Space.  
Более гибкая структура:

```python
# Пример создания узлов
boiler = server.nodes.objects.add_object(idx, "Boiler")
temp_node = boiler.add_variable(idx, "OutputTemp", 25.0)
level_node = boiler.add_variable(idx, "WaterLevel", 0.0)

temp_node.set_writable(True)


Загрузка тегов из CSV:
```python
import csv
from opcua import ua

# server_main.py
with open("tags.csv", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        name = row["name"]
        value = float(row["value"])
        writable = row["writable"].lower() == "true"
        
        node = boiler.add_variable(idx, name, value)
        if writable:
            node.set_writable()
```
Структура tags.csv:
```csv
name,value,writable
OutputTemp,25.0,True
WaterLevel,0.0,False
ValveHotIn,0.5,True
```
## Недостатки OPC UA
1. Сложность внедрения (выше порог входа)
2. Производительность чуть ниже OPC DA в простых случаях
3. Управление сертификатами требует квалификации
4. Ограниченная поддержка legacy-систем (нужны шлюзы)
5. Ресурсоёмкость при большом количестве узлов

## OPC UA для реального времени
OPC UA over TSN — для жёсткого реального времени
PubSub модель (MQTT / UDP) — для распределённых систем
