OPC UA Practice - модель процесса, OPC-сервер и Web-SCADA
Учебный проект, демонстрирующий полный цикл промышленной системы диспетчеризации:
Модель технологического процесса → OPC UA сервер → OPC UA клиент → Web-SCADA (Flask)
Идея проекта
Симулируется работа технологического. Модель на Python рассчитывает состояние объекта в реальном времени и публикует значения (теги) через OPC UA сервер. Любой OPC UA клиент может подключиться к серверу и читать/писать эти теги - в нашем случае это web-приложение на Flask, которое рисует показания и позволяет управлять процессом через браузер, как настоящая SCADA-система.

OPC UA (Open Platform Communications Unified Architecture)
Что такое OPC
OPC - это набор стандартов для обмена данными между промышленным оборудованием (ПЛК, датчики, приводы) и программным обеспечением (SCADA, MES, ERP). Стандарты разрабатывает и поддерживает некоммерческая организация OPC Foundation.
Изначально (1990-е) OPC расшифровывался как OLE for Process Control и был жёстко привязан к технологиям Microsoft. Позже, с появлением OPC UA, аббревиатуру переосмыслили как Open Platform Communications, потому что новый стандарт стал кроссплатформенным.
Зачем нужен OPC UA?
До появления единых стандартов каждый производитель оборудования писал свой протокол обмена данными, и подключение SCADA-системы к оборудованию разных вендоров превращалось в написание десятков разных драйверов. OPC решает эту проблему: оборудование и ПО общаются через единый интерфейс - не важно, кто производитель.

Архитектура клиент-сервер
•	OPC UA сервер - публикует данные технологического процесса (значения тегов, состояния, тревоги) в виде адресного пространства узлов.
•	OPC UA клиент - подключается к серверу, читает и/или записывает значения, подписывается на изменения (Subscriptions), вызывает методы.
В нашем проекте: opc_server.py - это OPC UA сервер, который "поднимает" на определённом порту модель процесса из boiler_model.py. Web-приложение на Flask (или отдельный скрипт-мост) выступает OPC UA клиентом, забирая данные для отображения в браузере.

Реализация в Python
Для Python есть библиотеки:
•	python-opcua - классическая синхронная 
•	asyncua - асинхронный форк той же библиотеки, актуальнее, рекомендуется для новых проектов.
Как это применяется в нашем проекте
1.	boiler_model.py каждую секунду пересчитывает состояние объекта.
2.	opc_server.py публикует эти значения как переменные OPC UA сервера.
3.	Web-SCADA (Flask) подключается как OPC UA клиент (или читает уже сохранённые данные из истории) и отображает их пользователю, а также может отправлять команды обратно на сервер (например, изменить положение клапана).
