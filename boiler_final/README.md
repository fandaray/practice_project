# Boiler SCADA — итоговый проект (сборка лучших решений студентов)

Симуляция бойлера смешения (горячая + холодная вода → выход) с публикацией
параметров через **OPC UA сервер** (порт 4840, как того требует задание) и
веб-панелью в духе промышленной SCADA/HMI-системы: анимированная мнемосхема,
авторегуляторы уровня/температуры, технологические блокировки, история в
SQLite.

Проект собран из четырёх студенческих реализаций одного и того же задания.
Ничего не написано "с нуля" мимо их кода — каждый файл ниже указывает,
чьё решение взято за основу и что из чужого решения в него добавлено.

## 1. Откуда что взято

| Файл | Основа | Что добавлено / почему выбрано |
|---|---|---|
| `boiler_model.py` | **rikhelgof** — физика на энергобалансе (кДж), давление, авторегуляторы уровня/температуры с зоной нечувствительности, технологические блокировки с гистерезисом | **khabarov** — плавная динамика клапанов (travel time): клапан не переключается мгновенно, а едет к уставке с ограниченной скоростью |
| `opc_server.py` | **rikhelgof** — иерархическое адресное пространство (`Measurements/Setpoints/Alarms/Interlocks/Effective`), логика "регулятор пишет обратно в Setpoints" | Порт изменён на **4840** (по заданию), добавлены теги `ValveTravelTime`, `InputTempHot/Cold` |
| `opcua_client.py` | **rikhelgof** — динамическое разрешение `namespace index` через `get_namespace_index()` вместо хардкода (`2:...`), что делает клиент устойчивым к порядку запуска серверов | **golubeva** — функция `validate_value()`, обрезающая присланные значения по допустимому диапазону перед записью |
| `history_logger.py` | **rikhelgof** — схема таблицы и общий подход опроса раз в N секунд | **golubeva** — автоматическая очистка (retention) устаревших записей и компактная сводка в лог вместо сырого словаря |
| `app.py` (Flask) | **rikhelgof** — REST API (`/api/state`, `/api/setpoint`), автопереподключение к OPC UA при обрыве связи, thread-safe клиент | Переиспользует общий `opcua_client.py` вместо собственного встроенного клиента |
| `templates/index.html`, `static/css/style.css`, `static/js/scada.js` | **rikhelgof** — анимированная SVG-мнемосхема (поток в трубах, уровень воды, цвет по температуре), тёмная промышленная тема | Добавлены элементы управления температурами магистралей и временем хода клапана (под новые теги) |
| `config.py` | **golubeva** — конфигурация через `.env`/переменные окружения вместо хардкода | — |

Решение **mamaev** было изучено (идея "запоминания" последнего стабильного
состояния клапанов при аварии), но не вошло напрямую в сборку — вся
необходимая защита уже покрывается блокировками (interlocks) rikhelgof.

> Почему основой стал именно rikhelgof: на момент сборки это единственное
> решение, целиком реализующее весь стек в одном стиле (модель → OPC UA
> сервер → клиент → логгер → веб-панель) с продуманной физикой и рабочим,
> уже протестированным фронтендом. Остальные решения дали точечные, но
> ценные улучшения (плавная динамика клапанов у khabarov, аккуратная
> конфигурация и валидация у golubeva).

## 3. Адресное пространство OPC UA

Endpoint: `opc.tcp://localhost:4840/boiler/server/`

| Узел | Тип | Доступ | Описание |
|---|---|---|---|
| `Boiler/Measurements/Level` | Double | RO | Уровень воды, % |
| `Boiler/Measurements/Temperature` | Double | RO | Температура смеси, °C |
| `Boiler/Measurements/TemperatureValid` | Boolean | RO | `False`, если бак пуст и мерить нечего |
| `Boiler/Measurements/Pressure` | Double | RO | Давление, бар |
| `Boiler/Setpoints/HotValve` | Double | RW | Уставка клапана горячей воды, % |
| `Boiler/Setpoints/ColdValve` | Double | RW | Уставка клапана холодной воды, % |
| `Boiler/Setpoints/OutletValve` | Double | RW | Уставка сливного клапана, % |
| `Boiler/Setpoints/Heater` | Double | RW | Мощность нагревателя, % |
| `Boiler/Setpoints/ValveTravelTime` | Double | RW | Время полного хода клапана, сек |
| `Boiler/Setpoints/InputTempHot` | Double | RW | Температура горячей магистрали, °C |
| `Boiler/Setpoints/InputTempCold` | Double | RW | Температура холодной магистрали, °C |
| `Boiler/Setpoints/LevelSetpoint` | Double | RW | Целевой уровень для авторегулятора, % |
| `Boiler/Setpoints/LevelAuto` | Boolean | RW | Включить авторегулирование уровня |
| `Boiler/Setpoints/TemperatureSetpoint` | Double | RW | Целевая температура для авторегулятора, °C |
| `Boiler/Setpoints/TemperatureAuto` | Boolean | RW | Включить авторегулирование температуры |
| `Boiler/Alarms/LevelHigh` \| `LevelLow` \| `OverTemperature` | Boolean | RO | Предупредительная сигнализация |
| `Boiler/Interlocks/Overflow` \| `Overtemp` | Boolean | RO | Сработавшие технологические блокировки |
| `Boiler/Effective/HotValve` \| `ColdValve` \| `Heater` | Double | RO | Фактическое положение (после травел-тайма и блокировок) |

## 4. Начальные условия (строго по заданию)

| Параметр | Значение |
|---|---|
| Уровень бака | 0 % (пустой) |
| Клапан горячей воды | 50 % |
| Клапан холодной воды | 50 % |
| Клапан выхода | 100 % |
| Температура горячей магистрали | 90 °C |
| Температура холодной магистрали | 10 °C |
| Шаг обновления модели | 1 секунда |

## 5. Как это работает

1. `opc_server.py` (через `server_main.py`) каждую секунду:
   - читает уставки клапанов/нагревателя/температур из адресного пространства;
   - если включён авторегулятор уровня или температуры — модель сама
     пересчитывает нужные положения клапанов/мощность нагревателя;
   - клапаны физически "едут" к уставке с ограниченной скоростью
     (`ValveTravelTime` секунд на полный ход 0→100%), а не переключаются
     мгновенно;
   - технологические блокировки (interlocks) проверяются и при
     необходимости принудительно закрывают приток (переполнение) или
     отключают нагреватель (перегрев) — независимо от того, кто задавал
     положение клапана: оператор или авторегулятор;
   - публикует новые измерения, аварии и фактические (`Effective`)
     положения исполнительных механизмов.
2. `app.py` — сам по себе OPC UA **клиент**: читает те же теги по HTTP-запросу
   браузера (`GET /api/state`) и пишет уставки обратно
   (`POST /api/setpoint`), когда пользователь двигает бегунок на панели.
3. `history_logger.py` — независимый OPC UA клиент, который пишет срез
   параметров в SQLite каждые 5 секунд и раз в ~100 циклов удаляет записи
   старше `HISTORY_RETENTION_HOURS` часов.

## 6. Установка и запуск без Docker

```bash
cd boiler_final
python3.11 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env             # необязательно, значения по умолчанию уже верны
```

Запуск в трёх отдельных терминалах (все команды — из папки `src/`):

```bash
cd src

# Терминал 1: OPC UA сервер с моделью бойлера
python server_main.py

# Терминал 2 (опционально): логгер истории в SQLite
python history_logger.py

# Терминал 3: веб-панель SCADA
python app.py
```

Панель откроется на **http://localhost:5050**.

Быстрая проверка связи без веб-панели:

```bash
cd src
python opcua_client.py
```
выведет текущий срез уровня/температуры/давления/клапанов.

## 7. Запуск через Docker

Одиночный контейнер (только OPC UA сервер, как того требует базовое задание):

```bash
docker build -t boiler-scada .
docker run --rm -p 4840:4840 boiler-scada
```

Полный стек (сервер + веб-панель + логгер истории) одной командой:

```bash
docker compose up --build
```

- OPC UA сервер: `opc.tcp://localhost:4840/boiler/server/`
- Веб-панель: **http://localhost:5050**
- История: файл `./data/boiler_history.sqlite3` на хосте (примонтирован как том)

Остановить: `docker compose down`.

## 8. Проверка, что всё работает

```bash
cd src
python -c "
from boiler_model import BoilerModel
m = BoilerModel()
print('Начальные условия:', m.state.level_pct, '%,', m.state.temperature_c, '°C')
for _ in range(60):
    m.step(dt_s=1.0)
print('Через 60 сек:', round(m.state.level_pct, 1), '%,', round(m.state.temperature_c, 1), '°C')
"
```

Ожидаемо: уровень и температура за 60 секунд плавно меняются в соответствии
с балансом клапанов 50/50/100% (уровень будет держаться низким, так как
слив открыт на 100% — приток не успевает за оттоком; чтобы увидеть
наполнение, прикройте `OutletValve` через клиент или веб-панель).
